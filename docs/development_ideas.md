# Идеи для дальнейшего развития проекта

## 1. Качество данных и аугментаций
- Добавить поддержку конфигурации аугментаций через YAML, чтобы гибко переключаться между наборами трансформаций под разные датасеты.
- Реализовать автоматический подсчёт статистик (среднее/стандартное отклонение) по обучающей выборке для нормализации вместо фиксированных значений ImageNet.
- Включить возможность балансировки классов через WeightedRandomSampler или вычисление весов классов при инициализации модели.

## 2. Модели и обучение
- Поддержать дообучение нескольких архитектур с общим интерфейсом (EfficientNet, ConvNeXt), что потребует расширения `create_model` и конфигураций.
- Интегрировать стратегии fine-tuning: заморозка частей сети, постепенное размораживание, использование LoRA для всех поддерживаемых моделей.
- Добавить автоматический подбор гиперпараметров (например, через Optuna) поверх уже имеющегося механизма sweeps.

## 3. Отслеживание экспериментов и воспроизводимость
- Интегрировать логирование в Weights & Biases или MLflow с сохранением гиперпараметров, метрик и артефактов.
- Сохранять в репозитории шаблоны для отчётов об эксперименте (markdown/jupyter) с визуализациями кривых обучения.
- Расширить автоматические проверки: статический анализ (ruff/mypy) и unit-тесты для ключевых модулей (датамодуль, модели, утилиты).

## 4. Продакшен и инференс
- Подготовить скрипты для экспорта моделей в ONNX/TorchScript и проверки совместимости.
- Реализовать REST/gRPC сервис для инференса с простым клиентом.
- Добавить smoke-тесты инференса (краткий прогон на нескольких изображениях) в pipeline CI.

Эти шаги помогут структурировать развитие и улучшить качество кода, воспроизводимость и готовность модели к продакшену.
